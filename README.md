# ArchiMate Viewer

Десктопное и веб-приложение для просмотра и редактирования моделей **ArchiMate** в формате XML (файлы `.archimate` и совместимый обменный XML). Интерфейс ориентирован на работу с диаграммами, деревом модели, локальным Git-репозиторием и проверками качества модели.

## О проекте

**ArchiMate Viewer** парсит XML-модель, строит внутреннее представление элементов, связей и диаграмм и отображает выбранную диаграмму на канвасе. Поддерживаются два распространённых представления файла:

- **Формат Archi** — нативные файлы приложения [Archi](https://www.archimatetool.com/) (структура с папками `folder` и типами `diagrams` / `relations`).
- **Обменный формат** — XML со структурой `model` → `elements`, `relationships`, `views` / `diagrams` / `view` (идентификаторы в атрибутах вроде `identifier`, геометрия узлов через `bounds` с атрибутами `w` и `h`).

Парсинг и сериализация обратно в XML реализованы в `src/lib/archimate/` так, чтобы сохранять совместимость с исходным форматом файла после правок (перемещение и размер объектов на диаграмме, изломы связей, свойства элементов, создание и удаление сущностей).

Для сценариев «модель в Git» поднимается локальный **Express**-сервер (`server/git-api.mjs`), который выполняет операции с репозиторием на машине разработчика: клонирование, чтение/запись файлов модели в пределах заданного корня, вспомогательные git-команды. В режиме разработки Vite проксирует запросы с префикса `/api` на этот сервер (см. `vite.config.js`).

Десктоп-оболочка реализована на **Wails v2** (Go). При запуске приложения Wails поднимает встроенный git-api (Node subprocess), а UI подключается к нему через локальный HTTP base URL.

## Основные возможности

### Моделирование

- Загрузка модели из файла и работа с деревом: элементы, связи, диаграммы; поиск по дереву.
- Канвас диаграммы: выбор объектов и связей, перетаскивание, изменение размеров, точки излома связей (добавление двойным щелчком, удаление клавишей **Delete** / **Backspace** или через UI).
- Создание новых объектов на диаграмме и режим создания связей между двумя объектами.
- Панель свойств: имя, документация, пользовательские `property`, навигация по диаграммам, где встречается элемент.
- Сохранение изменений обратно в XML с учётом правок и удалений.

### Git

- Выбор корня репозитория, клон по URL (в т.ч. с PAT для HTTPS), ветки, коммит и push (через локальный API и установленный `git` в PATH).
- Автозагрузка и запись файла модели по относительному пути внутри репозитория.

### Сравнение изменений

- Визуальное сопоставление текущего состояния диаграммы (включая несохранённые правки на канвасе) с версией из Git для выбранной диаграммы.

### Линтеры

- Набор проверок целостности модели (например, неиспользуемые на диаграммах элементы, несвязанные сущности); запуск по одной проверке или всех сразу. Реализации в каталоге `linters/`.

### Режим просмотра

- Упрощённый сценарий навигации по модели и диаграммам без полного набора инструментов редактирования.

### Активы и администрирование

- Зарезервированные разделы интерфейса; при необходимости сюда можно развивать отдельный функционал.

## Технологии

| Слой | Стек |
|------|------|
| UI | React 19, TypeScript, Vite 5, кастомный CSS |
| Локальный API | Node.js, Express |
| Десктоп | Wails v2 (Go), Node.js (git-api) |

На машине для Git-операций должен быть доступен исполняемый файл **`git`**.

## Требования

- **Node.js** (рекомендуется актуальная LTS-версия, совместимая с зависимостями проекта).
- **npm** (или совместимый менеджер пакетов).
- **Go** 1.21+ и **Wails CLI v2** (`go install github.com/wailsapp/wails/v2/cmd/wails@latest`) — для десктопной сборки.

## Установка

```bash
npm install
```

## Запуск в разработке

Полный режим — UI и git-api одновременно (API по умолчанию на порту **5151**):

```bash
npm run dev
```

Только фронтенд (без локального Git API; запросы к `/api` не будут работать без отдельно запущенного `dev:api`):

```bash
npm run dev:ui-only
```

Только сервер Git API:

```bash
npm run dev:api
```

После `npm run dev` откройте в браузере адрес, который выведет Vite (обычно `http://localhost:5173`).

## Сборка и десктопное приложение

Сборка статики:

```bash
npm run build
```

Просмотр production-сборки через Vite:

```bash
npm run preview
```

Разработка десктопного окна (Vite HMR + git-api из Go):

```bash
npm run dev:wails
```

Подготовка production-frontend + bundled git-api:

```bash
npm run build:desktop
```

Сборка десктопного приложения:

```bash
npm run build:mac
npm run build:win
```

Артефакты Wails по умолчанию попадают в `build/bin/`.

## Переменные окружения (git-api)

Переменные читает `server/git-api.mjs` (фрагменты документированы в начале файла):

| Переменная | Назначение |
|------------|------------|
| `GIT_REPO_ROOT` | Абсолютный путь к корню git-репозитория на диске (по умолчанию — родитель каталога `server`, т.е. корень проекта при локальной разработке). |
| `GIT_API_PORT` | Порт HTTP-сервера API (по умолчанию **5151**). |
| `GIT_API_HOST` | Хост привязки (в десктопном приложении задаётся `127.0.0.1`). |
| `SERVE_STATIC` / `STATIC_DIR` | Включение раздачи собранного UI из каталога (используется упакованным приложением). |

## Структура репозитория (кратко)

| Путь | Описание |
|------|----------|
| `src/App.tsx` | Состояние модели, вкладки, сохранение XML, интеграция с Git |
| `src/components/` | UI: канвас, боковая панель, Git, сравнение, линтеры и др. |
| `src/hooks/` | Хуки, в т.ч. `use-git-integration.ts` |
| `src/lib/archimate/` | Парсинг XML, геометрия, отрисовка, сравнение диаграмм |
| `server/git-api.mjs` | Локальный REST API для Git и файлов модели |
| `main.go`, `app.go`, `wails.json` | Точка входа Wails, запуск git-api |
| `internal/apiserver/` | Запуск встроенного git-api (Node) |
| `linters/` | Правила линтеров модели |
